# MacZoomer feature demos

This folder holds the animated GIFs embedded in the top-level `README.md`.

## Recording a demo

1. **Launch MacZoomer** and arrange your screen so the feature you're
   demonstrating is the focal point.
2. **Trigger recording** of just the relevant region:
   - `⌘⇧5` for a tight region (preferred — keeps file sizes down)
   - `⌘⌥5` for a single window
   - `⌘5` for the full display (avoid unless necessary)
3. **Perform the feature** for 4-10 seconds. Less is more — a GIF is meant to
   be a glance, not a manual.
4. **Stop** via the floating recording HUD's stop button, or by pressing the
   same hotkey again.

The MP4 lands in `~/Movies` (configurable in Settings → Recording).

## Converting to GIF

From the repo root:

```sh
scripts/mp4-to-gif.sh ~/Movies/MacZoomer\ Recording*.mp4 docs/demos/<feature>.gif
```

Optional overrides:

```sh
# Wider, smoother GIF — bigger file
scripts/mp4-to-gif.sh in.mp4 out.gif 960 24

# Tiny preview GIF
scripts/mp4-to-gif.sh in.mp4 out.gif 480 12
```

The script does a two-pass `palettegen` / `paletteuse` so colors stay
faithful with much smaller files than naive single-pass conversion.

with the real recording — keep the same filename and the README picks it
up automatically.

## File size guidance

Keep each GIF under **2 MB** if possible. If you exceed that, try:

- Reduce `WIDTH` first (640 is fine for README displays)
- Reduce `FPS` (12 is acceptable for most app demos)
- Shorten the clip — cut to the action with `ffmpeg -ss <start> -to <end>`
